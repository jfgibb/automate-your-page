Lesson 2: Introduction to Serious Programming

What is Computer Science
  • Computer Science is about how to solve problems by breaking the down into smaller pieces
  • Then define a sequence of steps that you can precisely solve each smaller problem
  • The steps must be able to be executed by a computer
  • Programing is the core of Computer Science
  • Focus on how to break down problems and not solving them in one big sweep
  • A computer is a multi-purpose device (it can do many things), not like a toaster that has a very focused purpose
  • But a computer canâ€™t do anything without a program
  • A program tells the computer what to do
  • A program must be very systematic and precise because a computer at its heart is "dumb" and only does what its told to by the program
  • Computers can execute programs very very fast
The Python Programming Language
  • Python is one of many programming languages availalbe
  • It is a high level language and relatively easy to use compared to lover level languages
  • This stage will use Python to introduce programing and the many concepts common to programing
  • The programs we will create will run on the Python program
  • Python defines a set of syntax and rules that must be followed by the programer for the computer to run the program
  • The Python program will take the high level statements in the program and convert them to the low level language the computer understands
  • This function is called an interperter
  • The following quote from Dave contains 3 of the 5 ways we can think like a programmer "The programs you'll write in this class will be Python code. Those will be input to another program which is a Python interpreter that follows the instructions in your code and it does that by following the instruction in its code... and you'll be able to run all that using your web browser."

  • The entire quote demonstrates a deep technological empathy (which you don't need to have at this point) for how computers and programs work
  • When Dave thinks of a Python program, a Python interpreter, and a web browser as different versions of the same thing (a computer program) he's showing abstract thinking
Starting Python
  • Python code is entered into an editor or a command line
  • The "Print" command will print something to the screen
  • The format of Print is Print <expresion>
  • The expression will be printed to the screen
  • Expresions can be simple (a word) or complex (results of a math equestion)
Why Not Just use Normal Language to Program
  • It would seem simpler to program a computer using natual language like Mr. Scott tries to in Star Trek IV
  • But natural languages are ambiguous and a computer needs precise instructions
  • Natural languages are very verbose; in a programming language a complex item can be defined in a few lines
  • A programing language will allow us to tell a computer what to do in a very precise way
Python Rules and Grammer
  • The Python interpreter only understands the rules of the python language, if you do something not supported it will create an error when you run the program
  • Python has a grammar like a natural language does
  • The difference in Python is you have to follow the grammar where in natural language you can not follow the grammar and someone may still be able to understand
  • Grammar based on Backus-Naur form:
  • Created by John Backus - leading developer of Fortran in the 1950s
  • Form included a statement of code with a comment on what that line did
  • First ways to program were not precise enough and this form helped programs be more precise
  • Backus-Naur was invented to precisely describe exactly the language in an easy to understand and also be very concise
  • Due to this Python grammar is much more precise than english grammar

Lesson 3: The Building Blocks of Python

Basic Operators and Expessions
  • An Expression is something that has a value
  • Two expressions can be combined with an operator to create a new expression
  • Some examples of python operators are +, -, *, and /
  • Example, Print 5 + 5 will result in 10 being printed
  • Numbers in python can be differnt types, integers and floats are two examples
  • An integer will not have decimal point while a float will
  • Its important to understand what type you are working with or the result may not be what you expect
  • For example, multipling two integers will result in an integar but multipling an integar and float will result in a float
Varriables
  • The varriable is one of the most important concepts in computer programming
  • A varriable is a place to store different data types (numbers, strings) within the program and to change them (hence the name varriable) as needed
  • Varriables are created using a varriable name, the assigment operator (assigment uses the = sign), and then an expression
  • The varriable name refers to the value of the expression that was assigned
  • An example: My_Name = 'John'
  • My_Name if the varriable name and John if the value it holds
  • Varriable Names should be descriptive of what they hold
  • Its important to note the = sign in Python does not mean equal like it does in math, it means assignement. The == means equal in Python
  • Varriables must be initiallized before they are used in a program. This means they have to have something assigned to it even if that value is nothing (empty string)
Strings and String Indexing
  • A String is a sequence of characters surrounded by single quotes
  • Example: 'John'
  • The + operator can be used to concatinate two strings
  • Example: 'Jo' + 'hn' becomes 'John'
  • A number can be a string if it has quotes around it '22' but you cannot add it to an integer if it is
  • Indexing allows us to pull out parts of a string based on the position of that part within the string
  • Each character within a string has a position number or index
  • The first character is a position 0, the second at position 1 and so on
  • 'John' - J is at position 0
  • Use the [ ] to pull out the character at the position
  • 'John[2] will output 'h'
  • If the string is assigned to avarriable the varriable name can be used
  • My_Name = 'John' My_Name[3] would output 'n'
  • A negitive index will search from the end of the string. My_Name[-1] will give 'n'
  • An index range can be give to get more than one character. My_Name[1:2] would give 'on'
  • The .find method is a built in program in python to ind a sub-string within a larger string
  • Format: <sting>.find(search term)
  • The output is the starting location (index) of the search term, if the search term is not found the output is -1
  • The # symple is to create a comment in a python program (the interperter will ignore the line
  • The triple quotes """ is used to create multi-line comments

Lesson 4: Functions, Conditionals, and Loops

Functions
  • Functions are smaller sub-sections of a program that take input from another part of the program, run some code, and send output back to the part of the program that made the call of the function
  • They allow the program to be broken down into small sub-programs and also allow a certain thing to be coded once and accessed each time its needed from the larger program
  • Functions are the key thing that allows for complex programs to be created
  • Functions are all about taking things as input and producing some output
  • Function syntax: <function name> (input, input, ...)
  • The inputs are the varriables passed into it and there can be any number of them but the call must pass in that many inputs
  • Names of the inputs do not have to be the same as the names used in the calling section of the program
  • To pass the result back to the calling section of code the return word is used: <return>
  • The code that executes and the varriables used inside a function are not visable outside the function
  • Functions can call other functions
Conditional Statements
  • Conditional statemens (if/then) are used in programs to evaluate an expression (test expression) and depending on the result take one action or another
  • A number of operators are available for the test expression including equal to (==), not equal to (!=), less than (<), and greater than (>)
  • If the restult of the test expression is True then the code that follows gets executed
  • If the test expression is False then there are a couple of options, one is to do nothing and just continue with the program and the second is to use the else statement to execture a different set of code
  • An example: if My_Name == 'John' print 'correct', else print 'incorrect'
  • The and and or booleans can be used in the if statement to execute the code if multiple test cases are all true (and) or if at least one of a list of test cases are true (or)
  • If statements can be nested within other if statements
Loops
  • Computers have to do many tasks over and over again and it would be difficult to code each itteration directly
  • Loops help solve this problem and allow repetive taksed to be done over and over again
  • A loop must have a way to exit the loop or it will run forever
  • Loops use test expressions just like if statements to determine if a loop should keep going or exit
  • Each time through the loop the test expression is tested to see if its True, if so the loop is exited and if not the loop keeps going
  • The same code is executed each time through the loop but the inputs may have changed
  • If statements can be used within a loop
  • The while loop is the first loop type we will look at
  • while (text expression is true):
execute code within the loop
  • The while loop usually uses a counter that gets incremented each time through the loop.
  • The counter will be compared to a value in the test expression and when the expression becomes false the program exists the loop
  • The break statement can be used to force an exit for the loop is a certain condition is met
Structured Data: Lists and For Loops
  • The string is the most basic form of structured data made up of a sequence of characters
  • We can operate down to each character within a string
  • A string has limits because it only holds characters
  • A list is a more complex data type because elements can be many different types (strings, ints, other lists)
  • List format: x=[a, b, c]
  • Indexing a list gives the value at the index position x[0] would be 'a'
  • An empty list can be created y = [ ]
  • The other big advantage of Lists over strings is the concept mutation
  • Mutation allows us to replace a certain element in the list
  • Mutation does not create a new list it actually changes the existing list
  • If two varriables point at the same list element and that element is mutated both varriables were changed
  • When a list is passed into a function the function operates on the origial list so changes are seen globally
  • the .append method can be used to append an element at the end of a list
  • The '+' operator will concatenate two lists
  • The len method returns the length of the list (note, if a list is an element in the outer list len does not count whats in the inner list)
  • The for loop can be used to cycle through all elements of a list and execute some code on each element
  • The format is: for <name> in <value>:
block of code
  • Each time through the loop the next index in the list will be assigned to name and the block of code execututed. This is done till all elements in the list have passsed via the loop

Lesson 5: Debugging and Problem Solving

Debugging
  • Bugs will happen during coding, there is no way around it
  • The important thing is to have a strategy to get rid of bugs
  • There are some bugs that will cause the program not to run by creating a traceback error
  • This is useful as it tells what type of error and what line it occured on
  • Other bugs will not cause an error but when the program runs the wrong output is created
  • These bugs are harder to notice and then find and fix
  • Using temporary print statements in different parts of the code can be very useful in debugging because you can look at varriables at different points
  • Testing of the code is vital to finding bugs
  • Test cases should incude a consise range of potential inputs and then looking to ensure the outputs are what is expected
Problem Solving
  • The main purpose of programming is to solve problems therefore it is a key skill to master
  • The first thing to do when solving a problem is to ensure you fully understand the problem else you may solve the wrong thing
  • Computational problems have inputs and desired outputs, we must understand each
  • All possible inputs are a set
  • The relationship between the inputs produce the desired output
  • Validate assumptions on the inputs which could limit the number of total inputs
  • Clearly define what the expected outputs are
  • Create some examples to test your inputs, outputs, and assumptions
  • Consider how to solve the problem in a systematic way as a human solves the problem
  • Next write Pseudocode before writing actual code:
  • Follows a programing style but is not tied to the exact rules of a language
  • Used to write out the algorithm to solve the proglem
  • Sometimes the pseudocode will not be the simplist way to solve the problem as the way humans think is not always the more efficient way to do something
  • Dave's Rules for Solving Problems:
  • Step 0: Don't Panic
  • Step 1: What are the inputs
  • Stap 2: What are the outputs
  • Step 3: Work through some examples by hand
  • Step 4: Find a simple and mechanical solution
  • Step 5: Develop the solution incrementally in small steps and test as you go
